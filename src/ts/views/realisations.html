Calendrier Logistique
Application web de calendrier de disponibilités client.
Technologies utilisées : JS, Jquery, OracleJet, NodeJS, OracleDB
Context :

Un de nos clients avait besoin d'une interface pour facilement consulter et mettre à jour la disponibilités de leurs fournisseurs.
Cette fonctionnalité n'existant pas dans l'ERP (JD Edwards) utilisé par ce client, ma mission était de développer une application web interfacée avec l'ERP. Ce projet a duré 9 mois.

Équipe :

Ce projet a été réalisé en binôme avec un consultant fonctionnel. Ce dernier ayant de très bonne connaissance sur le context du client et des paramétrages de L'ERP, il a pu me fournir un cahier des charges et des spécifications très détaillées.
Un administrateur système m'a également appuyé pour configurer le serveur web au sein de l'architecture réseau de l'entreprise.
Étant le seul développeur du projet, ce projet a été réalisée en autonomie, et a nécessité beaucoup de recherches et d'autoformations sur des technologies très peu utilisé dans mon service.

Fonctionnement :

Cette application web était accessible depuis le client web de l'ERP, où elle était intégrée à la page de consultation des fournisseurs.
Elle a été conçue avec le framework JS d'Oracle, Oracle JET, et avec la librairie FullCalendar.
Elle utilise l'API REST de JD Edwards pour consulter la base de donnée Oracle de l'ERP.
Le serveur web installé est un serveur NodeJS.

Commpétences :

Ingénierie du logiciel, Développement Web, Base de Données, Autonomie, Curiosité, Initiative

Factory
Appli tablette à destination d'opérateur sur une chaîne de production.
Technologies utilisées : Js, Typescript, OracleJet, Cordova, NodeJS, OracleDB, Git
Context :

Un de nos client avait besoin que les opérateurs de leurs lignes de production puissent avoir un accès facilité à l'ERP JD Edwards, particulièrement afin de :
- consulter la liste des ordres de fabrications
- Déclarer leurs temps
- Déclarer des incidents et des problèmes de qualité
La solution retenue par ce client a été une application hybride sur tablette.
Notre service n'ayant auparavant jamais accepté ce type de projet, il a été pensé comme un "projet école" afin de former le maximum de consultants à l'utilisation de ces technologies.
Débuté en Janvier 2020, ce projet est toujours en cours.

Équipe :

Ce projet a débuté avec une équipe nombreuse : nous étions 10 développeurs, dont 7 alternants, plus un consultant fonctionnel chargé de la tenue du cahier des charges, et notre chef de projet.
Étant parmi des développeurs les plus confirmés sur ces technologies, nous avions également à aider et former les développeurs plus débutants.
Ce nombre a rapidement baissé à 4-5 développeurs après les premières livraisons.

Fonctionnement :

Cette application hybride consiste en plusieurs écrans correspondants aux besoins cités plus haut, plus des écrans permettant de visualiser l'ensemble des lignes de production, à destination des managers.
Elle a été développée en Typescript, utilisant le framework d'Oracle OracleJet, et Cordova pour la déployer en application hybride sur les tablettes android du client.
Elle peut consulter la base de donnée Oracle via une API REST réalisée en NodeJS.
Un système de notifications en cas d'incidents a été mis en place, afin d'alerter les équipes concernées.
