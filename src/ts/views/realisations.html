Calendrier Logistique
Application web de calendrier de disponibilités client.
Technologies utilisées :

JS, Jquery, OracleJet, NodeJS, OracleDB

Contexte :

Un de nos clients avait besoin d'une interface pour facilement consulter et mettre à jour les disponibilités de leurs fournisseurs.
Cette fonctionnalité n'existant pas dans l'ERP (JD Edwards) utilisé par ce client, ma mission était de développer une application web interfacée avec l'ERP. Ce projet a duré 9 mois.

Équipe :

Ce projet a été réalisé en binôme avec un consultant fonctionnel. Ce dernier ayant de très bonne connaissance des activités du client et donc des paramétrages spécifiques de l'ERP, il a pu me fournir un cahier des charges et des spécifications très détaillés.
Un administrateur système m'a également appuyé pour configurer le serveur web au sein de l'architecture réseau de l'entreprise.
Étant le seul développeur du projet, ce projet a été réalisée en autonomie, et a nécessité beaucoup de recherches et d'autoformations sur des technologies très peu utilisées dans mon service.

Fonctionnement :

Cette application web était accessible depuis le client web de l'ERP, où elle était intégrée à la page de consultation des fournisseurs.
Elle a été conçue avec le framework JS d'Oracle, Oracle JET, et avec la librairie FullCalendar.
Elle utilise l'API REST de JD Edwards pour consulter la base de données Oracle de l'ERP.
Le serveur web installé est un serveur NodeJS.

Compétences :

Ingénierie du logiciel, Développement Web, Base de Données, Autonomie, Curiosité, Initiative.

Factory
Appli tablette à destination d'opérateurs sur une chaîne de production.
Technologies utilisées :

Js, Typescript, OracleJet, Cordova, NodeJS, OracleDB, Git

Contexte :

Un de nos clients avait besoin de fournir aux opérateurs de leurs lignes de production un accès facilité à l'ERP JD Edwards, particulièrement afin de :
- Consulter la liste des ordres de fabrications,
- Déclarer leurs temps,
- Déclarer des incidents et des problèmes de qualité.
La solution retenue par ce client a été une application hybride sur tablette.
Notre service n'ayant auparavant jamais accepté ce type de projet, ce dernier a été pensé comme un "projet école" afin de former le maximum de consultants à l'utilisation de ces technologies.
Débuté en Janvier 2020, ce projet est toujours en cours.

Équipe :

Ce projet a débuté avec une équipe relativement importante : nous étions 10 développeurs, dont 7 alternants, plus un consultant fonctionnel chargé de la tenue du cahier des charges, et notre chef de projet.
Étant parmi des développeurs les plus confirmés sur ces technologies, nous avions également à aider et former les développeurs plus débutants.
Ce nombre a rapidement baissé à 4-5 développeurs après les premières livraisons.

Fonctionnement :

Cette application hybride consiste en plusieurs écrans correspondants aux besoins cités plus haut, plus des écrans permettant de visualiser l'ensemble des lignes de production, à destination des managers.
Elle a été développée en Typescript, utilisant le framework d'Oracle OracleJet, et Apache Cordova pour la déployer en application hybride sur les tablettes android du client.
Elle peut consulter la base de données Oracle via une API REST réalisée en NodeJS.
Un système de notifications en cas d'incidents a été mis en place, afin d'alerter les équipes concernées.

Compétences :

Ingénierie du logiciel, Développement Web, Base de Données, Gestion de Projet, Curiosité.

Bubble Up
Application web de partage collaboratif de comics.
Technologies utilisées :

PHP, MySQL

Contexte :

Comme projet de formation, nous devions créer une page web en PHP.
Nous avons décidé de créer une plateforme de partage et de lecture de bande dessinée, où chaque utilisateur pourrait consulter le contenu existant, ou créer un compte pour téléverser son propre contenu.

Équipe :

Ce projet a été réalisé par trois étudiants développeurs.

Fonctionnement :

La base de données MySQL réference le contenu téléversé par chaque auteur enregistré.
Sur l'application web, chaque utilisateur a accès au contenu déjà validé par les administrateurs, et peut les visionner.
Il doit cependant se connecter pour pouvoir ajouter du contenu.
Entièrement responsive, l'interface a été pensée pour les appareils mobiles.
Ce projet a été réalisé en suivant la méthode SCRUM.

Compétences :

Ingénierie du logiciel, Développement Web, Base de Données, Gestion de Projet, Curiosité.

Big Browser
Application web de quiz de culture géographique.
Technologies utilisées :

PostgreSQL, Java, graphQL, Springboot, C#, XAML, Android

Contexte :

Comme projet de fin de formation, nous voulions créer un quiz basé sur des coordonnées géographiques.
L'utilisateur devait répondre à des questions de type "où se situe cette ville / ce monument?" ou "quelle est la ville de naissance de cette personne?" en cliquant sur une carte.

Équipe :

J'ai réalisé ce projet en binôme avec un autre étudiant développeur, géographe de formation initiale.

Fonctionnement :

L'application est composée :
- d'une base de données PostgreSQL, particulièrement intéressante pour stocker des données géographiques,
- d'une API Java GraphQL réalisée avec le framework Springboot,
- d'une application desktop XAML d'administration de la base de données, permettant de créer, supprimer et mettre à jour la liste des questions qui pourront être posées aux utilisateurs,
- et du client android contenant l'interface du jeu.

Compétences :

Ingénierie du logiciel, Algorithme et Structure de Données, Base de Données, Initiative, Curiosité.

Dungeon: The Quest of Sir Beback
Jeux d'aventure 2D en pixel art.
Technologies utilisées :

OpenSuse, C

Contexte :

Comme projet de fin d'année, nous voulions créer un jeu vidéo d'aventure avec quelques aspects RPG (gestion d'inventaire, progression).
L'histoire suivait un chevalier en armure combattant des monstres et résolvant des énigmes.
Étant fortement limités dans les technologies autorisées pour ce projet, nous l'avons réalisé avec une librairie graphique minimaliste, donc entièrement en pixel art et en 2D.
Un système de sauvegarde avait été ajouté.

Équipe :

J'ai réalisé ce projet en binôme avec un autre étudiant développeur, qui s'est notamment chargé de la réalisation des sprites en pixel art.

Fonctionnement :

Le jeu a été créé sous linux (OpenSuse), l'interface (lancement du jeu, selection de la sauvegarde) se fait en ligne de commande.
La librairie audio FMod nous a permis d'ajouter de la musique à chaque niveaux du jeu.
Nous avions également codé un pilote pour pouvoir brancher une manette et jouer avec.

Compétences :

Ingénierie du logiciel, Algorithme et Structure de Données, Initiative, Curiosité.

DobberMiam
Site web de partage de recettes de cuisine.
Technologies utilisées :

MongoDB, NodeJs, REST, React

Contexte :

Une part non négligeable de mes intéractions avec les autres membres de ma famille (nombreuse) consiste à tester, goûter et partager des recettes de cuisine.
Des notes griffonnées sur un bout de papier aux répertoires pleins de recettes numérisées, faire le tri et organiser ces recettes récupérées au fil des ans peut s'avérer complexe.
(Et frustrant lorsque l'on cherche précisément CETTE recette, reçue il y a quelques mois et bien sauvegardée ... mais où ?)
Le projet d'une plateforme de partage des-dites recettes s'est donc très vite imposé de lui-même.
Ce projet est actuellement en cours.

Équipe :

Je suis seul développeur sur ce projet, mais n'étant pas graphiste, j'ai notamment laissé à ma petite sœur le soin de dessiner le logo, et la charge de décider de la charte graphique.

Fonctionnement :

Un utilisateur connecté pourra consulter, commenter et noter les recettes présentes sur le site. Il pourra également poster les siennes.
La recherche de recette pourra être effectuée à partir de mot-clefs, ou à partir d'ingrédients précis (ou en excluant certains ingrédients).
Un utilisateur pourra également disposer d'une liste de recettes favorites, facilement accessibles.
La base de données a été initialement conçue en SQL (PostgreSQL), mais n'ayant auparavant jamais réalisé de projet en NoSQL, j'ai décidé de la refaire avec MongoDB.
L'API est réalisée avec NodeJS et Fastify.

Compétences :

Développement Web, Algorithme et Structure de Données, Initiative, Autonomie, Curiosité.

D&D Apps : Spellbook & Combat Calculator
Applications mobile de support au jeu de rôle.
Technologies utilisées :

JavaScript, OracleJet, Cordova, NodeJS

Contexte :

Le jeu de rôle en général et Donjons & Dragon en particulier sont depuis longtemps parmi mes passions.
Mais ce type de jeu peut paraître complexe aux débutants, surtout pour la quantité d'information à retenir et manipuler.
Par exemple, gérer un personnage de mage nécessite souvent beaucoup de feuilles volantes pour garder le compte des sorts disponibles, leurs descriptions, etc ...
De même, un joueur débutant gérant un personnage combattant pourra facilement se perdre dans la multiplicité des bonus et modificateurs spécifiques lors des combats.
Ces deux applications mobiles sont là pour aider à simplifier les actions, notamment en réduisant le temps passé à chercher des informations, ou calculer un bonus.

Équipe :

Je suis seul développeur sur ce projet.

Fonctionnement :

Ce sont deux applications hybrides, développées en Javascript avec les framework OracleJet et Apache Cordova.
L'application SpellBook permet de consulter une liste de sorts, d'en afficher les détails, et de filtrer la liste selon différents critères.
L'application Combat Calculator calcule automatiquement différents bonus selon de nombreux critères.

Compétences :

Développement Web, Algorithme et Structure de Données, Autonomie
